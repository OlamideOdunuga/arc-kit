5 minutes

From zero to
running app.

No docs required. Follow these steps and you'll have a working Arc App Kit project running locally by the end.

1
Install Node.js

You need Node.js 18 or later. Check if you already have it:

$ node -v

If you see a version number you're good. If not, download it at nodejs.org.

2
Get a Kit Key from Circle Console

arc-appkit uses Circle's App Kit SDK under the hood. You need a free Kit Key to make any capability work.

Go to console.circle.com → create an account → create a new project → copy your Kit Key.

Testnet Kit Keys are free. No credit card needed.
3
Scaffold your project

Run the CLI and answer two questions — project name and capability.

$ npx arc-appkit create
? What is your project name? my-arc-app
? Which capability? Send

A new folder is created with everything pre-wired.

4
Fill in your environment variables

Inside your new project folder, copy the env file:

$ cd my-arc-app
$ cp .env.example .env.local

Open .env.local and fill in your Circle API Key and wallet private key.

Never commit .env.local — your private key lives here. It's already in .gitignore.
5
Install and run

Install dependencies and start the dev server:

$ npm install
$ npm run dev

Open localhost:3000 — your Arc app is running.

6
Get testnet USDC

Your app is running but you need testnet USDC to actually use it. Head to the Circle faucet and request some to your wallet address.

faucet.circle.com — free testnet USDC, no signup needed.

You're live on Arc Testnet.
