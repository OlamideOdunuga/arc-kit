Documentation

arc-appkit is a CLI and React component library that scaffolds working Arc Network projects — built on top of Circle's official App Kit SDK.

Arc is in public testnet. @circle-fin/app-kit requires a free Kit Key from Circle Console — get one before you start.

Prerequisites

Before using arc-appkit, make sure you have:

  • Node.js 18 or later
  • A free Circle Console account, for your Kit Key
  • A wallet with testnet USDC — get some from the Circle faucet

Installation

Pick the path that matches what you're building. New project — use the CLI. Existing project — install the library.

$ npx arc-appkit create
$ npm install @arc-appkit/react

CLI — Usage

Run the create command and answer two prompts:

$ npx arc-appkit create
? What is your project name? my-arc-app
? Which capability? Bridge / Swap / Send / Unified Balance

That's it — a complete Next.js project is generated in a new folder.

What gets generated

my-arc-app/
├── app/
│   ├── layout.js
│   ├── page.js          ← capability UI
│   ├── globals.css
│   └── api/[capability]/route.js   ← SDK call
├── .env.example
├── .gitignore
├── next.config.js
├── README.md
└── package.json

The UI lives in app/page.js. The actual blockchain call — using @circle-fin/app-kit — lives server-side in the API route, so your private key never touches the browser.

Library — Install

Already have a project? Drop components in directly:

$ npm install @arc-appkit/react

Components

import { ArcBridge, ArcSwap, ArcSend, ArcUnifiedBalance } from '@arc-appkit/react'

export default function Page() {
  return <ArcSwap kitKey={process.env.NEXT_PUBLIC_KIT_KEY} />
}
PropTypeDescription
kitKeystringYour Circle Console Kit Key

Each component renders its own UI and calls the corresponding API route in your app — see each capability below for what to set up on the backend.

Capabilities

Four building blocks, each wrapping a different part of the App Kit SDK.

Bridge Swap Send Unified Balance

Bridge

Transfers USDC from another chain (e.g. Ethereum Sepolia) onto Arc Testnet using Circle's CCTP-based bridging.

POST /api/bridge

Swap

Exchanges USDC for EURC on Arc Testnet in a single call.

POST /api/swap
{ "amount": "10.00" }

Send

A direct wallet-to-wallet USDC transfer on Arc.

POST /api/send
{ "recipient": "0x...", "amount": "5.00" }

Unified Balance

Reads a single combined USDC balance across all chains the App Kit supports — no manual multichain aggregation needed.

GET /api/balance

Configuration

Every generated project ships with a .env.example. Copy it to .env.local and fill in:

VariableDescription
CIRCLE_API_KEYYour Kit Key from Circle Console
PRIVATE_KEYWallet private key used server-side to sign transactions
NEXT_PUBLIC_ARC_RPChttps://arc-testnet.drpc.org
NEXT_PUBLIC_ARC_CHAIN_ID5042002
Never commit .env.local. It's already in .gitignore by default.

Deployment

Generated projects deploy to Vercel like any Next.js app:

$ npm install -g vercel
$ vercel

Add your environment variables in the Vercel dashboard under Project Settings → Environment Variables before your first deploy.

FAQ

SWC binary fails to load on Windows

Reinstall the platform binary directly: npm install @next/swc-win32-x64-msvc --force

"Scope not found" when installing

Make sure you're on npm 8+ and have a clean internet connection — scoped packages like @arc-appkit/react sometimes fail silently on flaky networks.

Where do I get testnet USDC?

The Circle faucet issues testnet USDC directly to your wallet address.
